## 🔄 AUTO-DEPLOYMENT (Magic Part)

### How It Works:

```

1. You push code to GitHub
   git push

2. GitHub notifies Vercel

3. Vercel automatically:
   ├─ Pulls latest code

   ├─ Runs: npm run build
   ├─ Deploys new version
   └─ Updates live URL

4. Your live app is updated!
   No manual steps needed ✅

All in 2-3 minutes ⏱️
```

## 🆘 TROUBLESHOOTING

### App Deployment Failed

```
If you see ❌ Failed:

1. Check build logs
   Click project → "Deployments" → latest → View logs

2. Common issues:
   ├─ Missing environment variables
   │  Fix: Add to Vercel dashboard
   │
   ├─ Package installation failed
   │  Fix: Check package.json
   │
   ├─ TypeScript errors
   │  Fix: Run locally: npm run build
   │
   └─ Port already in use
      Fix: Vercel handles this, should work

3. Still failing?
   Redeploy: Dashboard → Deployments → Redeploy
```

### API Not Working

```
If API calls fail on live URL:

1. Check environment variables
   Settings → Environment Variables
   Verify API keys are there

2. Check CORS settings
   Backend should allow all origins (already set)

3. Check logs
   Vercel → Deployments → Logs
   Look for errors

4. Test locally first
   npm run dev
   http://localhost:3000/api/stocks/AAPL
```

### Can't Access Dashboard

```
If you forget password:

1. Go to: https://vercel.com/login
2. Click: "Forgot password?"
3. Enter email
4. Check email for reset link
5. Create new password
6. Login ✅
```
