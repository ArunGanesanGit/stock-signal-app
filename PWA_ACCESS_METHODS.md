## 💻 TECHNICAL DETAILS FOR USERS

### How PWA Access Works Behind the Scenes

```
┌─────────────────────────────────────────────────┐
│          PWA ACCESS ARCHITECTURE                │
├─────────────────────────────────────────────────┤
│                                                 │
│  User's Device                                 │
│  ├─ Browser (Safari/Chrome)                   │
│  ├─ Service Worker (installed on device)      │
│  │  └─ Caches app locally                     │
│  └─ Home Screen (if installed)                │
│                                                 │
│  Vercel Server                                │
│  ├─ Hosts: Next.js app                        │
│  ├─ Serves: HTML/CSS/JS                       │
│  └─ API: Express backend                      │
│                                                 │
│  Data Flow:                                    │
│  User Opens App                                │
│    ↓                                           │
│  Service Worker checks: Is cached version ok? │
│    ├─ YES → Load from cache (instant!)       │
│    └─ NO → Fetch from Vercel server          │
│    ↓                                           │
│  Display app to user                          │
│    ↓                                           │
│  User clicks "Analyze AAPL"                  │
│    ↓                                           │
│  Fetch data from: /api/signals/AAPL          │
│    ↓                                           │
│  Vercel backend calls: AlphaVantage API       │
│    ↓                                           │
│  Get real data, display to user               │
│                                                 │
└─────────────────────────────────────────────────┘
```
